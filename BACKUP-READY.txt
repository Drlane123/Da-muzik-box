WHY CURSOR BLOCKS YOUR SOURCE ZIP
==================================

It is NOT blocking the zip file on E:\.
Cursor locks FILES INSIDE your project while it is open.

What happened (2026-06-14 backup at 2:08 PM):
  tar ran for ~40 minutes, reached 2 GB, then failed with:

  "Couldn't open ./.cursor/workspace-data/terminals/235686.txt: Permission denied"

That file is Cursor's own terminal log for the backup command.
Cursor keeps it open. tar cannot read it. tar stops. Zip never finishes.

Same problem can happen with:
  - .cursor/workspace-data/terminals/*.txt
  - .cursor/workspace-data/agent-transcripts/
  - any file Cursor is actively writing to

FIX (already applied in scripts/pack-source-zip.ps1):
  Backup now SKIPS .cursor/workspace-data (Cursor temp data, not source code).
  Your app code, .git, rules, scripts are still included.
